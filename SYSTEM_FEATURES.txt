1. SYSTEM OVERVIEW
================================================================================

DistroGH is a consignment distribution platform for managing vendors who supply
products through DistroGH to supermarkets. The system tracks:

  - Vendor products, pricing, and stock received at the warehouse
  - Deliveries from DistroGH to supermarket branches
  - Monthly supermarket sales imports
  - Returns and deductions
  - Vendor balances and mobile-money (MoMo) payouts
  - Compliance (FDA certificates, service charges, verification)

Currency: Ghana Cedis (GHS)

================================================================================
2. USER ROLES & ACCESS
================================================================================

ADMIN
  Full access to vendors, products, sales import, deliveries, payouts, reports,
  settings, applications, and deactivation requests.

VENDOR
  Portal for self-service vendors: dashboard, products (own), sales (own),
  returns (own), receiving visibility, payouts, statements, delivery status,
  profile, FDA upload, service charge, deactivation request.

USER
  Limited staff role: dashboard, sales view, supermarkets, store stock, reports.
  (No vendor management, payouts, deliveries, or sales import.)

Access modes for vendors:
  - self_service   — vendor logs in and manages their own portal
  - admin_managed  — DistroGH admin operates on behalf of vendor; vendor sees
                     a read-only "vendor report" style statement

7. PRODUCTS
================================================================================

Products page (/dashboard/products) — Admin & Vendor
  - List all products (vendor sees own only)
  - Create / edit / soft-delete products
  - Product fields: name, vendor, vendor price, DistroGH markup, selling price,
    SKU, barcode, category, packaging size, wholesale price, supermarket
    selling price, MOQ, expiry date, product images
  - Live integrity checks on save:
      * Barcode globally unique (blocks duplicate)
      * Name + SKU combination unique (blocks duplicate)
      * Same SKU on different product names allowed (info)
      * Same name with different SKU allowed (info)
  - Pagination and search
  - Relative "last updated" timestamps

Categories (Settings)
  - Admin manages product categories used in product forms

Stock API

9. DELIVERIES TO SUPERMARKETS
================================================================================

Deliveries page (/dashboard/deliveries) — Admin only
  - Create delivery run: supermarket, date, transport cost, notes, line items
  - Stock validation (cannot deliver more than on hand)
  - Preview vendor transport charge split when creating run
  - List delivery runs with filters (supermarket, date range)
  - Pending vs confirmed status
  - Confirm delivery workflow:
      * Review/edit transport cost before confirm
      * Edit per-vendor charge split or auto "split by quantity"
      * Balance check: charges must equal transport cost
      * On confirm: supermarket inventory updated
      * Transport cost split across vendors by units delivered
      * Vendor deductions created automatically (linked to delivery run)
      * Deductions reduce vendor payout balance
  - View applied charge split on confirmed runs (lazy-loaded)
  - Sidebar badge for pending (unconfirmed) deliveries

Delivery charge allocation rules:
  - Default: quantity-weighted split of total transport cost
  - Manual override per vendor before confirm
  - One shared vehicle → multiple vendors pay proportional to units on run
  - Zero transport cost → no deductions

11. SALES
================================================================================

Sales list (/dashboard/sales) — Admin, Vendor, User
  - View sales records with filters
  - Monthly period display (sales stored as calendar months)
  - Month filter
  - Vendor-scoped view for vendor role
  - Shows qty sold, revenue, commission, vendor due per line

Sales import (/dashboard/sales/import) — Admin only
  - Upload Excel spreadsheet from supermarket
  - Select reporting supermarket (chain) and report month
  - Parse rows: product, quantity, branch, dates, etc.
  - Match products by name/SKU/barcode
  - Match supermarket branches within selected chain
  - Outlet picker when spreadsheet has no BRANCH column
  - Preview matched / unmatched / duplicate rows
  - Create missing products or supermarkets inline during import
  - Bulk insert validated sales (normalized to full calendar month)
  - Download sample Excel template
  - Import draft saved in browser session (resume later)
  - Import history page

Sales calculations (per line):
  - Revenue = qty × selling price
  - Vendor due = qty × vendor price
  - Commission = revenue − vendor due (DistroGH margin)

14. PAYOUTS & VENDOR BALANCES
================================================================================

Payouts page (/dashboard/payouts) — Admin only
  Tabs:
    Vendor balances
      - All vendors with positive balance owed
      - Total outstanding footer
      - Duplicate open payout detection and cleanup banner
      - Create payout for individual vendor
      - Bulk create payouts for all vendors with balance

    Pending payments
      - Open payout records awaiting MoMo payment
      - Record partial or full payment
      - MoMo transaction ID tracking (append multiple)
      - Status: pending → partial → completed

    Payment history
      - Completed and past payouts

  Features:
    - Week/period range on payout records
    - Prevent duplicate open payouts per vendor (DB constraint)
    - Race-safe payment recording (transaction + row lock)
    - Reuse existing open payout if one already exists
    - Sidebar alert badge for pending payout count
    - Pagination on all tabs

Vendor balance formula:
  Balance = sales vendor due − returns − deductions − completed payout amounts

Vendor payouts portal (/dashboard/vendor/payouts) — Vendor

20. DATA INTEGRITY & BUSINESS RULES
================================================================================

  - Soft delete on vendors, products, payouts, intakes, returns
  - Unique barcode per product (after deduplication migration)
  - Unique name+SKU per product (after deduplication migration)
  - Delivery confirm is transactional (inventory + charges + confirm flag)
  - Idempotent delivery vendor charges (no double deduction per run)
  - Payout open-record uniqueness per vendor
  - Sales periods stored as full calendar months
  - Transport charges validated: sum of vendor shares = total cost
  - Server-side stock check on delivery creation
  - PostgreSQL reporting views for vendor balances
